🛡️ HSE Scoring System

Manual de usuario y referencia técnica

v1.0 | Última actualización: Marzo 2026

📖 Introducción

El HSE Scoring System es un modelo de calificación para evaluar contratistas en materia de Salud, Seguridad y Medio Ambiente (HSE). Funciona de manera similar a un scoring bancario de crédito, donde:

  • Se evalúan múltiples indicadores de riesgo
  • Se calcula una puntuación de confianza basada en el historial
  • Se asigna un nivel de escalamiento según la matriz 5x5
  • Se genera un plan de remediación con checkpoints
💡 Analogía bancaria: Así como un banco evalúa el riesgo crediticio y define límites de crédito, este sistema evalúa el riesgo HSE y define niveles de supervisión y restricciones operativas.

📊 Sistema de Scoring

Risk Score (0-100)

Mide el nivel de riesgo basado en indicadores negativos:

  • Tasa de eventos: Incidentes por cada 10,000 HH trabajadas
  • Acciones vencidas: Cantidad y criticidad de acciones sin cerrar
  • Cierres pendientes: Meses sin realizar cierre mensual
  • Tendencia: Comparación con meses anteriores
Risk Score = (Event Rate × 25) + (Overdue × 15) + (Missing Close × 20) + (Trend × 10)

Trust Score (0-100)

Mide la confiabilidad basada en indicadores positivos:

  • Calidad de datos: Completitud y consistencia de reportes
  • Historial de cumplimiento: Meses sin incidentes
  • Respuesta a hallazgos: Tiempo de cierre de acciones

Celda 5x5

La combinación de Risk y Trust determina la posición en la matriz:

Risk Score Nivel Trust Score Nivel
0-20R1 (Muy Bajo)0-20T1 (Muy Baja)
20-40R2 (Bajo)20-40T2 (Baja)
40-60R3 (Moderado)40-60T3 (Media)
60-80R4 (Alto)60-80T4 (Alta)
80-100R5 (Crítico)80-100T5 (Muy Alta)

📐 Matriz de Escalamiento 5x5

La matriz combina Riesgo (R1-R5) en el eje vertical con Confianza (T1-T5) en el eje horizontal para determinar el nivel de escalamiento (1-5).

📊 Visor interactivo: Abre matrix_viewer.html para explorar la matriz de forma visual.

Interpretación de la Matriz

🔴 Esquina R5-T1

Peor escenario: Alto riesgo + Baja confianza = Suspensión inmediata

🟢 Esquina R1-T5

Mejor escenario: Bajo riesgo + Alta confianza = Proveedor estratégico

🟡 Diagonal

Balance: El riesgo se compensa con la confianza (supervisión estándar)

⚠️ Anomalías

R1-T1: Bajo riesgo pero poca visibilidad = Regularizar documentación

🌐 Multilenguaje (i18n)

El sistema soporta español (es), inglés (en) y portugués (pt).

Cambiar idioma en Python

from config import set_language, get_label, get_current_language

# Ver idioma actual (default: "es")
print(get_current_language())  # "es"

# Cambiar a inglés
set_language("en")

# Obtener label traducido
print(get_label("risk_levels", "R5"))  # "Critical"

# Cambiar a portugués
set_language("pt")
print(get_label("risk_levels", "R5"))  # "Crítico"

Agregar un nuevo idioma

  1. Editar SUPPORTED_LANGUAGES en config/loader.py
  2. Agregar traducciones en el JSON para cada label
"R5": {
  "es": "Crítico",
  "en": "Critical", 
  "pt": "Crítico",
  "fr": "Critique"  ← Agregar nuevo idioma
}

⚠️ Sistema de Penalizaciones

Similar a los intereses moratorios en banca, el sistema aplica penalizaciones que crecen con el tiempo sin mejora.

Cómo funcionan

  • Acciones críticas vencidas: +3 pts por acción
  • Backlog de acciones: +0.35 pts por acción no crítica
  • Meses sin cierre: +5 pts por mes
  • Crecimiento logarítmico: La penalización crece más lento con el tiempo
Penalty = base × log(1 + months_overdue)

Decay por buen comportamiento

Las penalizaciones decaen un 15% por mes de buen comportamiento:

if is_good_month:
    penalty *= (1 - DECAY_RATE)  # 15% decay
⚠️ Importante: Las penalizaciones se acumulan pero tienen un cap máximo para evitar scores desproporcionados.

📋 Planes de Remediación

Cada celda de la matriz define un plan de remediación con tareas y checkpoints. Al igual que un plan de pago en cuotas, no se espera mejora inmediata.

Estructura del plan

  • target_risk_reduction: % de reducción de riesgo esperada
  • target_trust_increase: % de aumento de confianza esperado
  • max_days_to_improve: Plazo total del plan
  • tasks: Lista de tareas priorizadas
  • checkpoint_days: Días para verificar avance

Checkpoints típicos

Plazo totalCheckpoints
90 días[30, 60, 90]
45 días[15, 30, 45]
21 días[7, 14, 21]
14 días[7, 14]

📈 Modelo Bayesiano

El sistema incluye un modelo Poisson-Gamma para predecir la probabilidad de incidentes en los próximos 30 días.

Prior Gamma

λ ~ Gamma(α, β)
α = 2.0 (shape) | β = 10.0 (rate)

Posterior

Después de observar k incidentes en t meses:

α' = α + Σk
β' = β + t

Predicción

from core.bayesian import BayesianIncidentModel

model = BayesianIncidentModel()
prob = model.predict_incident_probability_30d(
    observed_incidents=[2, 0, 1, 0, 3],  # Últimos 5 meses
    months_observed=5
)
print(f"P(incidente en 30d): {prob:.1%}")
📊 Nota: El modelo requiere scipy para cálculos avanzados. Sin scipy, usa una aproximación simplificada.
